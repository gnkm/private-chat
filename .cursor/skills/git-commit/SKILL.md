# Git コミット（Conventional Commits・日本語）

## 前提

- メッセージのルールは [Conventional Commits 1.0.0](https://www.conventionalcommits.org/ja/v1.0.0/) に従う。
- **タイトルと本文は日本語**で書く（型・スコープ部分のキーワードは慣例どおり英語のままでよい。例: `feat`, `fix`, `docs`）。

## 手順

1. **変更の把握**
   - `git status -sb` を実行する。
   - ステージ済みなら `git diff --cached`、未ステージなら `git diff` で内容を確認する。
2. **ステージング**
   - **コミット対象はステージ済みの差分を既定とする。** ステージが空で未ステージの変更だけがある場合は、ユーザーに `git add` の範囲を確認するか、ユーザーが「全部ステージして」と明示したときだけ広く `git add` する。推測で無関係ファイルをステージしない。
3. **メッセージ作成**
   - 1 行目: `<型>[任意 スコープ]: <日本語のタイトル>`
   - 型は変更内容に合わせる: `feat`, `fix`, `docs`, `chore`, `refactor`, `perf`, `test`, `ci`, `build` など（ブランチ命名規則の型と揃えられる）。
   - スコープは任意。パッケージ・領域が明確なら括弧で短く（例: `feat(cli): ...`）。
   - 補足が必要なら **1 行空けて**本文を日本語で書く。
   - 破壊的変更は型の直後に `!` か、本文・フッターに `BREAKING CHANGE:` を付ける（CONTRIBUTING のとおり）。
   - **1 コミットに無関係な変更を詰めない。** 差分が複数の論理単位に分かれるなら、ユーザーに分割コミットを提案する。
4. **コミット実行**
   - ステージ済みの内容に合ったメッセージで `git commit` する。
   - 一行タイトルのみ: `git commit -m "feat(scope): タイトル"`
   - 本文あり: `git commit -m "feat(scope): タイトル" -m "本文の1段落目"` のように複数 `-m` を使うか、`GIT_EDITOR` / 一時ファイルで複数行にする。

## メッセージ例（日本語タイトル）

```text
feat(cli): 行番号オプション -n を追加

マッチした行の先頭に 1 始まりの行番号を付与する。

docs: CONTRIBUTING のコミット例を整理する

fix(parser): 空ファイル入力時に落ちないよう修正する
```

## 注意

- コミット前にユーザーが別のメッセージ形式を指定した場合はそれに従う。
- シークレットやトークンをメッセージやコマンドに含めない。
